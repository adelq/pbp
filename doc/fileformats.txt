crypto output is always: nonce|c(msg)
signed output is always sig1[32]msg|sig2[32]

.sk - secret subkeys
nonce|cipher(cipher secret|signing secret)

.pk - public keys
sign(mk, master public | cipher public | sign public | created[32] | valid[32] | name... )

.mk - secret master key
nonce|cipher(master secret)

pfs keys are for each pear one file like this:
nonce|c(mykey|mynextkey|peerkey|peernextkey)

.sig - external signatures on public keys
calculate sign(mk,pk) -> sig1[32]|pk|sig2[32]
store sig1|sig2 appending to the .sig file.
